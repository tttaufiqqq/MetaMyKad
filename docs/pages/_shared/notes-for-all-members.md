# Notes For All Members

Status: Draft  
Type: Shared page rules

- Derived IC values should be shown as read-only outputs after parsing.
- Optional uploads must still show their current status clearly.
- Never hide search filters after submission; keep them sticky or re-rendered.
- Every result row should link to the student detail page.
- Deletion must never be a one-click action with no confirmation.
- On mobile, tables may stack into cards, but the key matched metadata must remain visible.
- Reuse the template's visual direction, but do not import React/Vite code into the live PHP app.
- Keep backend truth in PHP/MySQL, even if the template prototype had frontend-only demo logic.
